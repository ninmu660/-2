# 設計書

## 1. システム構成

本アプリは、HTML、CSS、JavaScriptのみで構成される静的Webアプリケーションである。バックエンドサーバーや外部APIを使用せず、ブラウザ上で全処理を完結させる。

```text
index.html
  ├─ data/movies.js
  ├─ src/app.js
  └─ src/styles.css
```

## 2. ファイル構成

### 2.1 `index.html`

- アプリのエントリーポイント。
- CSSとJavaScriptを読み込む。
- `#app` 要素を配置し、JavaScriptが画面全体を描画する。

### 2.2 `data/movies.js`

- 初期映画データを `window.seedMovies` として定義する。
- APIキーや外部データ取得なしで動作するためのローカルモックデータベースとして扱う。

### 2.3 `src/app.js`

- アプリケーションの状態管理、画面描画、イベント処理を担当する。
- 検索、絞り込み、並び替え、追加、削除、状態変更を実装する。
- Canvasによるポスター画像生成を行う。
- `localStorage` を使ってユーザー追加データを永続化する。

### 2.4 `src/styles.css`

- レイアウト、配色、レスポンシブ対応、カード表示、フォーム表示を担当する。

## 3. データ設計

### 3.1 映画データ

映画データは以下の構造を持つ。

```js
{
  id: "seed-0",
  title: "The Shawshank Redemption",
  year: 1994,
  director: "Frank Darabont",
  runtime: 142,
  rating: 9.3,
  genres: ["Drama"],
  country: "USA",
  status: "Watched",
  tags: ["hope", "prison", "friendship"],
  logline: "A quiet banker endures decades in prison...",
  cast: ["Tim Robbins", "Morgan Freeman"],
  colorA: "#243b55",
  colorB: "#f2a65a"
}
```

### 3.2 `id`

- 初期データは `seed-{index}` とする。
- ユーザー追加データは `custom-{timestamp}` とする。
- `custom-` で始まるデータのみ削除可能とする。

### 3.3 `status`

内部値は英語キーで保持する。

| 内部値 | 表示 |
| --- | --- |
| `Favorite` | お気に入り |
| `Watched` | 鑑賞済み |
| `To Watch` | 観たい |

絞り込み用に `All` を使用し、表示は `すべて` とする。

### 3.4 `genres`

ジャンルは内部値として英語キーを保持する。表示時は `genreLabels` によって日本語化する。

例:

| 内部値 | 表示 |
| --- | --- |
| `Drama` | ドラマ |
| `Sci-Fi` | SF |
| `Action` | アクション |

ユーザーが手動追加した未知のジャンルは、そのまま表示する。

## 4. 状態管理設計

アプリ全体の状態は `state` オブジェクトで管理する。

```js
const state = {
  movies: loadMovies(),
  query: "",
  genre: "All",
  status: "All",
  sort: "rating",
  selectedId: null
};
```

### 4.1 `movies`

- 初期映画データとユーザー追加データを結合した配列。

### 4.2 `query`

- 検索欄の入力値。

### 4.3 `genre`

- 現在選択中のジャンル絞り込み。

### 4.4 `status`

- 現在選択中の状態絞り込み。

### 4.5 `sort`

- 現在選択中の並び替え条件。

### 4.6 `selectedId`

- 詳細表示中の映画ID。

## 5. 永続化設計

### 5.1 保存先

ユーザー追加データはブラウザの `localStorage` に保存する。

```text
key: cinema-vault-local-v1
value: JSON文字列
```

### 5.2 保存対象

- `id` が `custom-` で始まる映画のみ保存する。
- 初期データは `data/movies.js` から毎回読み込む。

### 5.3 読み込み順

1. `data/movies.js` から初期データを読み込む。
2. `localStorage` からユーザー追加データを読み込む。
3. 両者を結合して `state.movies` に設定する。

## 6. 画面設計

### 6.1 ヘッダー

- アプリ名を表示する。
- 映画追加ボタンを配置する。

### 6.2 コントロールエリア

- 検索欄
- ジャンル選択
- 状態選択
- 並び替え選択

### 6.3 統計エリア

- 登録作品
- 表示中
- お気に入り
- 平均評価

### 6.4 一覧エリア

- 映画カードをグリッド表示する。
- カードをクリックすると詳細表示対象を切り替える。

### 6.5 詳細エリア

- 選択中映画の詳細を表示する。
- 状態変更ボタンを配置する。
- 手動追加映画の場合のみ削除ボタンを表示する。

### 6.6 追加ダイアログ

- `<dialog>` 要素で映画追加フォームを表示する。
- 入力後、アプリ状態と `localStorage` に反映する。

## 7. 処理設計

### 7.1 初期表示

1. `window.seedMovies` を読み込む。
2. 初期データに `seed-` IDを付与する。
3. `localStorage` の追加映画を読み込む。
4. 画面を描画する。

### 7.2 検索・絞り込み・並び替え

1. `getFilteredMovies()` を実行する。
2. 検索条件、ジャンル条件、状態条件で絞り込む。
3. 並び替え条件に従ってソートする。
4. 結果を一覧に表示する。

### 7.3 映画追加

1. 追加フォームの入力値を `FormData` で取得する。
2. 必要な型変換を行う。
3. `custom-` IDを付与する。
4. `state.movies` に追加する。
5. `localStorage` に保存する。
6. 追加した映画を詳細表示対象にする。

### 7.4 状態変更

1. 詳細画面の状態ボタンを押す。
2. 対象映画の `status` を次の状態へ変更する。
3. ユーザー追加データを保存する。
4. 画面を再描画する。

### 7.5 削除

1. 削除ボタンを押す。
2. `custom-` IDの映画のみ削除対象とする。
3. `state.movies` から除外する。
4. `localStorage` に保存する。
5. 選択状態を解除して再描画する。

## 8. ポスター生成設計

### 8.1 方針

公式ポスター画像や外部画像APIは使用しない。映画ごとの `colorA`、`colorB`、タイトル、年、監督、ジャンル、評価を使い、Canvasでポスター風画像を生成する。

### 8.2 生成内容

- グラデーション背景
- 装飾的な円形ライン
- ポスター枠
- タイトル
- 公開年と監督
- ジャンル
- 評価

### 8.3 出力形式

- Canvasから `data:image/jpeg` のData URLとして生成する。
- 生成結果を `<img>` の `src` に設定する。

## 9. UI設計

### 9.1 レイアウト

- 上部にヘッダー。
- その下に検索・絞り込み操作。
- 中央に統計。
- 下部を一覧と詳細の2カラム構成にする。
- 画面幅が狭い場合は1カラムに切り替える。

### 9.2 配色

- 映画アプリらしい暗色ヘッダーを採用する。
- 本文領域は読みやすい明色背景にする。
- ジャンルチップ、状態ボタン、危険操作ボタンを色で区別する。

### 9.3 レスポンシブ

- PCでは一覧と詳細を横並びにする。
- タブレット・スマートフォンでは縦並びにする。
- フォームは狭い画面で1列にする。

## 10. エラーハンドリング

- `localStorage` のJSON解析に失敗した場合は、初期データのみで起動する。
- 未知のジャンルは変換せず、そのまま表示する。
- 検索結果が0件の場合は、該当なしメッセージを表示する。

## 11. セキュリティ設計

- ユーザー入力を画面表示するときは `escapeHtml()` でHTMLエスケープする。
- 外部通信を行わないため、APIキーや認証情報は扱わない。
- 入力データはローカルブラウザ内にのみ保存する。

## 12. 今後の拡張設計

- データアクセス層を分離し、将来的にAPIやIndexedDBへ置き換え可能にする。
- 映画追加フォームを編集フォームとして再利用できる構造にする。
- ポスター生成処理をキャッシュし、映画数が増えた場合の描画負荷を抑える。
- 日本語タイトル、原題、メモ、視聴日などのフィールド追加に対応する。
